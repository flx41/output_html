## 一、项目概述

本项目是一个基于Web的A股技术分析工具，融合了**Al Brooks价格行为学**和**顾比均线法则（GMMA）**两大分析体系，提供K线图表展示、技术指标计算、交易信号生成等功能。

### 核心功能
- K线图表展示（日K/周K/月K/30分/60分）
- 顾比均线（GMMA）12条EMA显示
- 金叉/死叉自动识别
- 技术指标：MACD、RSI、KDJ、布林带、成交量
- 价格行为模式识别：Pin Bar、吞没形态、十字星
- 综合交易信号评分（价格行为60% + GMMA 40%）
- 中文名/股票代码搜索

## 三、核心算法

### 3.1 EMA（指数移动平均）

```javascript
function calcEMA(data, period) {
    const ema = [];
    const k = 2 / (period + 1);
    
    // 第一个EMA值使用SMA（简单移动平均）
    let sum = 0;
    for (let i = 0; i < period; i++) sum += data[i].close;
    ema[period - 1] = sum / period;
    
    // 后续EMA = (当前值 - 前一日EMA) * 系数 + 前一日EMA
    for (let i = period; i < data.length; i++) {
        ema[i] = (data[i].close - ema[i - 1]) * k + ema[i - 1];
    }
    return ema;
}
```

### 3.2 GMMA趋势分析

**均线分组：**
- 短期组：EMA 3, 5, 8, 10, 12, 15（反映短期投机者）
- 长期组：EMA 30, 35, 40, 45, 50, 60（反映长期投资者）

**趋势判断逻辑：**
```javascript
// 计算两组均线的平均值
shortAvg = 短期组6条均线的平均值
longAvg = 长期组6条均线的平均值

// 判断趋势
if (shortAvg > longAvg * 1.02) → 多头排列
else if (shortAvg < longAvg * 0.98) → 空头排列
else → 震荡整理
```

**金叉/死叉检测：**
```javascript
// 金叉：短期组从下向上穿越长期组
if (prevShortAvg <= prevLongAvg && currShortAvg > currLongAvg) → 金叉

// 死叉：短期组从上向下穿越长期组
if (prevShortAvg >= prevLongAvg && currShortAvg < currLongAvg) → 死叉
```

### 3.3 MACD计算

```javascript
// 参数：快线12，慢线26，信号线9
DIF = EMA(close, 12) - EMA(close, 26)
DEA = EMA(DIF, 9)
MACD柱 = DIF - DEA
```

### 3.4 RSI计算

```javascript
// 参数：周期14
RS = 平均上涨幅度 / 平均下跌幅度
RSI = 100 - (100 / (1 + RS))
```

### 3.5 KDJ计算

```javascript
// 参数：N=9，M1=3，M2=3
RSV = (收盘价 - N日最低价) / (N日最高价 - N日最低价) * 100
K = (2 * 前一日K + RSV) / 3
D = (2 * 前一日D + K) / 3
J = 3 * K - 2 * D
```

### 3.6 布林带计算

```javascript
// 参数：周期20，倍数2
中轨 = SMA(close, 20)
标准差 = sqrt(Σ(close - 中轨)² / 20)
上轨 = 中轨 + 2 * 标准差
下轨 = 中轨 - 2 * 标准差
```

### 3.7 综合信号评分

```javascript
// 价格行为得分（权重60%）
paScore = 根据Pin Bar、吞没形态数量计算，范围 -100 ~ 100

// GMMA得分（权重40%）
gmmaScore = 多头为正，空头为负，范围 -100 ~ 100

// 综合得分
totalScore = paScore * 0.6 + gmmaScore * 0.4

// 信号判断
if (totalScore > 30) → 做多信号
else if (totalScore < -30) → 做空信号
else → 观望
```

### 5.3 腾讯API字段坑

```javascript
// ❌ 错误：写死取.day字段
const klines = data.data?.['sh600519']?.day || [];

// ✅ 正确：按周期取对应字段
let klines;
if (period === '30' || period === '60') {
    klines = data.data?.[`${prefix}${code}`]?.[`m${period}`] || [];
```

**踩坑：** 周K返回 `week`，月K返回 `month`，分钟返回 `m30`/`m60`，不能统一用 `day`。

### 5.4 时间轴同步问题

```javascript
// ❌ 错误：使用timeRange同步，可能为null
chart.timeScale().subscribeVisibleTimeRangeChange(timeRange => {
    indicatorChart.timeScale().setVisibleRange(timeRange);  // 报错：Value is null
});

// ✅ 正确：使用logicalRange + 防抖
let isSyncing = false;
chart.timeScale().subscribeVisibleLogicalRangeChange(logicalRange => {
    if (isSyncing || !logicalRange) return;
    isSyncing = true;
    try {
        indicatorChart.timeScale().setVisibleLogicalRange(logicalRange);
    } catch (e) {}
    setTimeout(() => { isSyncing = false; }, 10);
});
```

**踩坑：** 
- `timeRange` 可能为 `null`，导致 Lightweight Charts 报错
- 两个图表互相订阅会形成循环触发，需要防抖

### 5.5 金叉死叉检测bug

```javascript
// ❌ 错误：reduce累加包含undefined值
const shortAvg = shortEmas.reduce((s, e) => s + (e[i] || 0), 0) / GMMA_SHORT.length;

// ✅ 正确：过滤undefined后再计算
function getAvg(arr, idx) {
    const vals = arr.map(e => e[idx]).filter(v => v !== undefined && v !== null);
    return vals.length > 0 ? vals.reduce((s, v) => s + v, 0) / vals.length : 0;
}
```

**踩坑：** EMA数组前几个值是 `undefined`，直接 `|| 0` 会把真实值为0的数据误判。

### 5.6 日期解析时区问题
